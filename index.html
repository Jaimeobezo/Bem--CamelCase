INVESTIGACIÓN Y APLICACIÓN DE METODOLOGÍAS BEM Y CAMELCASE

¿Qué es BEM?

BEM (Block Element Modifier) es una metodología de nomenclatura para clases en HTML y CSS que tiene como objetivo escribir código más claro, estructurado, mantenible y escalable en proyectos web.

Ventajas de usar BEM

  1. Claridad y legibilidad
  2. Estructura modular
  3. Evita conflictos en el CSS
  4. Escalabilidad
  5. Mejor colaboración en equipos

Ejemplos de código usando BEM

  • Menú de navegación con ítem activo
  • Tarjeta de producto
  • Mensaje de alerta
  • Formulario con campos de error
    • Grupo con dos campos lado a lado (ej. nombre y apellido)
  • Lista de tareas
  • Encabezado de sitio con navegación
  • Botón con modificadores

¿Qué es camelCase?

Es una convención para escribir nombres compuestos (identificadores) donde:

  • La primera palabra empieza con minúscula
  • Las siguientes empiezan con mayúscula
  • No se usan espacios ni guiones bajos

Situaciones donde se usa camelCase

  1. Nombres de variables en muchos lenguajes de programación
  2. Nombres de funciones o métodos
  3. Propiedades de objetos en JavaScript/JSON
  4. Nombres de parámetros en funciones
  5. Nombres de campos en bases de datos (en algunos estilos de ORM)

Ejemplos de código usando camelCase

  • lista de productos constante = ["pan", "leche", "huevos"]; → arreglo con nombre en camelCase
  • usuario constante = { primer nombre: "Xiomara", apellido: "Sifuentes" }; → objeto con propiedades en camelCase
  • funcion showAlertMessage() { alerta("¡Hola!"); } → función que muestra un mensaje
  • class ShoppingCart { addItem(product) { ... } } → método de clase en camelCase
  • const isAvailable = true; → booleano en camelCase
  • document.querySelector(".mainContainer"); → clase CSS con camelCase (válido pero no recomendado)
